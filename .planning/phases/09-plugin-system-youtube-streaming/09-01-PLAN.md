<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> `/api/plugins` | Browser sends plugin ids, config patches, and action requests into server runtime. |
| Plugin runtime -> config files | Server writes enabled/config state under `.local/plugins`. |
| Runtime/debug -> browser/logs | Server converts internal plugin state and errors into browser DTOs and debug artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | `src/server/routes/plugins.ts` | mitigate | Block completion if any plugin route response serializes OAuth tokens, refresh tokens, RTSP URLs, RTMP(S) URLs, stream names, client secrets, or camera credentials. Add route-level unsafe payload assertions and tests. |
| T-09-02 | Tampering | `src/config/plugin-config.ts` | mitigate | Validate all config reads and writes with Zod, write only parsed values, reload after write, and reject malformed config patches with 422. |
| T-09-03 | Elevation of Privilege | `src/plugins/plugin-registry.ts` | mitigate | Use static first-party registration only; do not load arbitrary filesystem paths, npm packages, or browser-provided plugin code. |
| T-09-04 | Repudiation | Plugin actions | accept | Single-user LAN-only app has limited audit needs in this phase; action results should still include safe state/status for troubleshooting. |
</threat_model>
