<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> YouTube plugin auth actions | Browser initiates config/auth/revoke but must never receive OAuth token material. |
| Local server -> Google OAuth endpoints | Server exchanges codes and refreshes/revokes tokens with Google. |
| Server -> `.local/plugins/youtube` | Server persists OAuth client config, state, and tokens. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-05 | Information Disclosure | `youtube-oauth.ts`, `youtube-config.ts`, `plugins.ts` | mitigate | Treat OAuth tokens, refresh tokens, client secrets, and auth codes as blocking secrets. Add redacted DTO builders and route/debug tests that serialize full payloads. |
| T-09-06 | Spoofing | OAuth callback state | mitigate | Generate and verify callback state before token exchange; reject missing/mismatched state with 400 and do not persist tokens. |
| T-09-07 | Tampering | OAuth client import/config | mitigate | Validate Desktop OAuth client JSON with Zod and reject unknown/malformed config through plugin action validation. |
| T-09-08 | Repudiation | Revoke/refresh actions | accept | Single-user LAN-only app does not need audit trails in this phase; return safe action result and updated auth status. |
</threat_model>
