<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server -> YouTube Live API | Server creates and transitions external broadcast resources. |
| Server -> camera RTSP source | Server reads credential-bearing local camera stream URL. |
| Server -> FFmpeg process | Server passes secret-bearing input/output URLs into child process arguments. |
| Browser -> stream actions | Browser can request setup/start/stop but must not receive ingestion or RTSP secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-09 | Information Disclosure | `youtube-stream-service.ts`, `youtube-ffmpeg-config.ts` | mitigate | Redact RTSP URLs, camera credentials, RTMP(S) ingestion URLs, stream names/keys, OAuth tokens, and process args before status, logs, tests, or debug artifacts. Blocking severity. |
| T-09-10 | Tampering | Broadcast start action | mitigate | Validate start body with Zod, require safe default privacy, and require explicit confirmation for public broadcasts to reduce accidental public broadcast risk. Blocking severity. |
| T-09-11 | Denial of Service | FFmpeg process lifecycle | mitigate | Keep singleton/idempotent process state; avoid spawning duplicates; stop process on user/shutdown/failure paths. |
| T-09-12 | Information Disclosure | YouTube Live API resource handling | mitigate | Treat `cdn.ingestionInfo` as secret-bearing; persist only resource ids and safe share metadata. |
</threat_model>
