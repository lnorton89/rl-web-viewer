<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser API client -> server plugin routes | Browser receives only DTOs and sends action requests. |
| Hook state -> rendered DOM | Hook data becomes visible to user and snapshots/tests. |
| Share controls -> clipboard/window | Browser copies or opens safe YouTube watch URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-13 | Information Disclosure | `YoutubeStreamingPanel.tsx` | mitigate | Render only safe DTO fields; add DOM serialization tests blocking tokens, client secrets, RTSP URLs, RTMP(S) URLs, stream names, and camera credentials. |
| T-09-14 | Tampering | Start/public controls | mitigate | Disable actions while prerequisites are missing or pending; require explicit confirmation for public broadcast start before invoking server action. |
| T-09-15 | Spoofing | Auth URL handling | mitigate | Treat auth URL as server-provided safe URL only; do not parse or display auth codes/tokens in the browser. |
| T-09-16 | Information Disclosure | Share controls | mitigate | Copy/open only YouTube watch URL; never expose ingestion endpoint or stream key as share data. Blocking severity. |
</threat_model>
